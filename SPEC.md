## ゲーム概要

コミカル路線のゾンビ護衛タワーディフェンス。  
護衛対象（家族）をスタートからゴールまで安全に送り届けるのが目的。  
タワーを設置してゾンビの群れを撃退する。

## 用語定義（混乱防止のため必読）

| 用語 | 定義 | 例 |
|------|------|-----|
| **WAVE** | 護衛キャラ1人がS→Gを完走する単位 | WAVE1=お父さん、WAVE2=お母さん |
| **スポーンフェーズ** | 各WAVE中のゾンビ出現強度の段階 | 序盤・中盤・終盤で敵の強さが変わる |

- **WAVEはエスコートの数だけある**（ステージ設計者が決める）
- **スポーンフェーズは各WAVEの内部設定**（JSON の `waves[]` フィールドが実体、将来的に `spawnPhases[]` へリネーム予定）
- ⚠️ コード内の `waves` / `WaveManager` はスポーンフェーズの実装。WAVEシステム（護衛リレー）とは別物

> コード改修時に「WAVE」という単語を見たら、護衛リレーかスポーンフェーズかを必ず確認すること

#### タッチ操作改善 ★最重要（モバイル品質直結）
- **ピンチ（ズーム）**: 実装済みだが動作要改善

## ステージ設計思想（2026-06-26 確定）

### 任意導線への移行で変わったこと

| 旧：FlowField自動経路 | 新：任意導線 |
|----------------------|-------------|
| プロップ = 通路の壁（経路を強制するため必須） | プロップ = 純粋な装飾・ゲームプレイ要素 |
| マップ設計 = 「どこを塞いで経路を作るか」 | マップ設計 = 「どんな経路を引くか」 |
| 護衛は常に最短経路を選ぶ | 護衛は設計者が意図した任意の経路を歩く |

### 解放されたデザイン自由度

- **壁なしで複雑なルート**が作れる（建物の裏を抜ける・公園を横断・大回りなど）
- **護衛の経路 ≠ ゾンビの経路**（ゾンビはFlowFieldで護衛を追跡、護衛は固定経路）
- **緩急のある導線**が自然に設計できる（路地→広場→路地 など）
- 経路を長くすれば **タワー設置の猶予**を作れる、短くすれば**緊張感**を高められる

### ステージ設計の指針

- 導線の長さ・形状がゲームの難易度に直結する
- プロップは「見た目の世界観」と「ゾンビの動線妨害」のためだけに配置する
- 将来の **寄り道システム(Y)** も同じpath配列内のwaypoint indexで管理する

## GPT生成プロンプトのコツ（2026-06-26 確定）

### 正面・背面スプライト生成時の必須指定

**視点：**
- 「真上から斜め下ろしの視点（トップダウン斜め）」を最初に明示
- 「頭頂部が見える角度」を明記

**アニメーション指示：**
- ❌ 「歩行モーションを描いて」→ 全フレームほぼ同じ足になる
- ✅ 「歩いている瞬間を5回ストップモーションで描画」と指示

**足のポーズを1枚ずつ明示：**
```
frame1: 右足前
frame2: 中間（直立）
frame3: 左足前
frame4: 中間（直立）
frame5: 右足前
```

→ AIが「5枚全部ほぼ同じ足」になる現象をかなり回避できる
