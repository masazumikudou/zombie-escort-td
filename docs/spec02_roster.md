# 仕様書02：家族リレー・ロースター（確定版 2026-06-12）

## コンセプト
- 護衛者は1ステージに複数人、順次リレー（前の人がゴール/脱落で次が出発）
- リレーは実質ウェーブ構造：後の護衛者ほど難しくなければならない
  （タワー資産が育っているため。尻すぼみ厳禁）
- ロースターは「パレット」：全員を毎ステージ出す必要はなく、
  出場メンバー・順番・勝利条件はステージごとに自由に設定

## 難易度設計の3レバー
1. ウェーブ側：後の護衛者ほど数・強さを盛る（基本）
2. 護衛者側：守りにくい個性の人を後ろに配置（出発順＝難易度曲線）
3. 地形側：後の護衛者のルートを変え、既存タワー資産を部分的に無価値化
   （※強烈なので中盤ステージから解禁。序盤は同ルートで文法を教える）

## ロースター（初期5名）
| メンバー | speed | HP | 個性（解き方が変わる差） |
|---|---|---|---|
| 父 | 80 | 100 | 基準キャラ。看板（任意誘導）に釣られる。チュートリアル担当 |
| 母 | 100 | 70 | 速い＝先回り配置が間に合わない。強制寄り道を高確率で踏む |
| 息子 | 120 | 50 | 最速＋蛇行移動（パニック走り）。先回りの読みが困難 |
| おばあちゃん | 45 | 130 | 最遅タンク。被弾機会最大。看板を無視（任意誘導が封じられる） |
| 猫 | 150 | 30 | ルール破壊枠：道を無視し障害物以外を直進。当たり判定極小。ボーナス/特殊面の主役 |

- 数値はすべて初期仮値。バランス調整項目
- ロースターは将来追加可能な構造にする（キャラ定義＝データ＋スプライト）

## JSONスキーマ
```json
"escorts": [
  {
    "variant": "dad",
    "start": { "col": 0, "row": 7 },
    "goal":  { "col": 19, "row": 7 },
    "hp": 100, "speed": 80,
    "startTrigger": "afterPrevious",
    "waves": [ ... ]
  }
],
"minSurvivors": 1
```
- 各護衛者が個別に start / goal / waves を持つ（ルート変更・往復を表現可能）
- startTrigger：現状 "afterPrevious" のみ実装。スキーマとしては
  { "delayMs": n } 形式を将来許容（同時進行護衛の布石。実装しない）
- HPは完全個別。共有なし

## リレー挙動
- N人目がゴール到達 or 脱落 → 約4秒のインターバル → N+1人目出発
- インターバル中：
  - 「{名前}、出発！」バナー表示
  - 時間は流れ続ける。タワー設置・売却は可能
  - 次の護衛者のルートをカメラが軽くなぞるプレビュー（ルート可変のため必須）
- 残存ゾンビは消さず持ち越し。次の護衛者出発時にターゲット自動切替

## 脱落・勝敗
- HP0＝死亡ではなく「脱落」：座り込み演出→離脱（コミカル調維持。死の絵は出さない）
- 脱落者はゾンビのターゲットから外れる
- クリア判定：全員消化後、生還数 >= minSurvivors
- 敗北判定：「現生還数＋未出発人数 < minSurvivors」成立の瞬間に即敗北
  （負け確の消化試合をさせない）
- 敗北演出：ブラックアウト→敗北イラスト（assets/sprites/ui/gameover.png、
  フォールバック方式）→ GAME OVER → リトライ YES/NO
- リザルト：星評価（生還数/総数）＋撃破数表示

## 拡張台帳（実装対象外）
- 同時進行護衛（startTrigger.delayMs）：終盤の難易度レバー候補
- 追加キャラ案：娘（息子と性能交換可）、おじいちゃん（ランダム逸脱、
  理不尽リスクのため要慎重検討）、犬・赤ちゃん等
