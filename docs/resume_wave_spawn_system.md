# WaveManager スポーンシステム レジュメ
## 別AIへの引継ぎ資料

> **注意：** 会話中に「すごろく方式」という言葉が出ることがあるが、これは開発者とCCの間だけの略称。さいころもすごろくも関係ない。正式名称は「**最寄りスポーン選択方式**」。

---

## ゲーム概要

ゾンビ護衛タワーディフェンス。プレイヤーは護衛対象（エスコート）をゴールまで安全に送り届けるためタワーを設置する。エスコートはA*経路探索で自動移動し、ゾンビはエスコートを追いかけてくる。

---

## スポーンシステムの設計思想（最寄りスポーン選択方式）

### 核心コンセプト
「エスコートが今いる場所」に最も近いスポーン地点から敵を出現させる。

すごろくのコマ（エスコート）の現在位置を基準として、スポーン地点の優先順位が**リアルタイムで変動**する。エスコートが前進するほど、後方のスポーンは無効化され、前方・隣接のスポーンが優先される。

### なぜこの方式か
- エスコートが通過した遠方から敵を出しても意味がない（追いつけない）
- 難易度は「エスコートの現在位置に合わせた圧力」で決まる
- スポーン地点をエディターで自由配置でき、ステージ設計の自由度が高い

## ルール詳細

### ① 最寄りスポーン選択
グループ発火タイミングで、全スポーン地点を「エスコートへのユークリッド距離」でソートし、最も近い1点を選択する。

```
距離 = √((spawn.col - escort.col)² + (spawn.row - escort.row)²)
```

グループ発火時に1回決定し、そのグループの全員が同じスポーン地点から出る（途中変更なし）。

### ② グループスポーン
4体が800ms間隔で1列に出現（隊列）。グループ間は7秒。

```
[t=0.0s] ゾンビ1体目
[t=0.8s] ゾンビ2体目
[t=1.6s] ゾンビ3体目
[t=2.4s] ゾンビ4体目
[t=9.4s] 次グループ1体目（7秒後）
```

グループ内のスポーン地点は固定。エスコートが移動しても今のグループは元の地点から出続ける。次グループの発火時に改めて最寄りを計算する。

### ③ 時間帯によるウェーブ切替
スポーン開始（初回グループ発火）を elapsed=0 として、duration 秒後に次のウェーブへ切り替わる。JSON記載の数値と実挙動が一致する。

```
[t=0s]   WAVE1 開始（startDelay後に最初のグループ発火）
[t=60s]  WAVE2 切替（duration=60）
[t=120s] WAVE3 切替（duration=60）
[t=∞]   WAVE3 継続（duration=9999）
```

## JSONパラメーター

`stages/stage_XX.json` の `escorts[n].waves[]` に記述する。

```json
{
  "startDelay": 6000,
  "duration": 60,
  "spawnInterval": 7000,
  "groupSize": 4,
  "groupInterval": 800,
  "enemy": {
    "type": "normal",
    "hp": 30,
    "speed": 55,
    "damage": 10,
    "reward": 20
  }
}
```

| パラメーター | 単位 | 説明 |
|---|---|---|
| `startDelay` | ms | waves[0]のみ有効。ゲーム開始から初回グループまでの待機 |
| `duration` | 秒 | このウェーブの継続時間（次ウェーブへの切替タイミング） |
| `spawnInterval` | ms | グループとグループの間隔（推奨7000）|
| `groupSize` | 体 | 1グループの人数（推奨4） |
| `groupInterval` | ms | グループ内の個体間隔（推奨800） |
| `enemy.type` | - | `normal` / `normal_cap` / `normal_helmet` / `alt` / `alt_cap` / `alt_helmet` |
| `enemy.hp` | - | HP直値（省略時はZOMBIE_BASEのデフォルト） |
| `enemy.speed` | - | 移動速度直値 |
| `enemy.damage` | - | 護衛への攻撃ダメージ |
| `enemy.reward` | - | 撃破時の報酬金額 |

## WaveManagerの内部状態

```
this.waveIdx         : 現在のウェーブインデックス（0始まり）
this.startTime       : elapsed=0 の基準時刻（startDelay後に設定）
this.nextGroupTime   : 次グループを発火する絶対時刻
this._groupRemaining : 現グループの残り出現数
this._groupNextTime  : グループ内次の1体の発火時刻
this._groupSpawn     : 現グループのスポーン地点（{col, row}）
```

## スポーン停止条件

以下のいずれかでスポーンが停止する：

1. **護衛死亡** `escort.alive === false`
2. **護衛ゴール到達** `escort.reached === true`

どちらも `update()` の先頭ガードで検出する。ゲーム終了処理（勝利・敗北モーダル）はGameScene側の `_checkWinLose()` が担当。

---

## 安全上限

`config.js` の `MAX_ZOMBIES = 120`。`_spawnZombie()` の先頭で `this.zombies.length >= MAX_ZOMBIES` なら `null` を返しスキップ。ゾンビが減れば自動再開。難易度調整用ではなくスマホクラッシュ防止のための安全網。
