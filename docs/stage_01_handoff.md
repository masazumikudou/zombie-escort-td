## 2. なぜこの設計が「正しい」のか（設計思想の核）

### 2-1. TD＝パズル。ランダム性を排除する

このゲームは**決定論的パズル**。同じ配置なら必ず同じ結果になる。ランダム湧きは「上級者が最適解を再現できなくなる」ため**絶対に採用しない**（小松さんの明確な設計思想）。

- 上級者: 交戦地を読み切り、完全解（100%）を再現
- 中級者: 各交戦地で1回被弾してもクリア（90%）
- 初級者: プレイヤー自身の未熟さで結果がブレる（ゲーム側のランダムではない）

### 2-2. 護衛hpは盛らない（却下された方向）

**「護衛hpを上げてクリアさせる」方向は明確に却下された。** それはマップの粗さをステータスで誤魔化す行為で、パズルとして不正。護衛hp=100は**固定の設計定数**であり、ステージごとにいじる変数ではない。

難易度は「敵数・敵種・マップ構造（置き場の制約）」で作る。ステータス盛りで作らない。

### 2-3. 「1交戦地1体」が基準

L字での即死問題（団子3体が1点に集まり30ダメ/秒で3.3秒即死）の根本解決。

- **1交戦地1体なら、ノーガードでも張り付き1体=10ダメ/秒**。即死しない
- 中級者が1体見逃しても致命傷にならない
- 上級者は1体でも先回り撃破でノーダメ
- **敵を複数出したいときは「同じ交戦地に団子」ではなく「別の交戦地に分散」または「隊列（後述）」で出す**

### 2-4. マップが敵を散らすと即死が消える

このマップは経路が長く（71区間）、交戦地が4つに分散し、一等地（1タワーで2交戦地カバー）が存在する。この構造自体が「敵を捌きやすく」している。L字の「1本道・団子・即死」と真逆。**難易度はマップ構造の締め付け（buildSpots制限・prop設置）で作るのが健全。**

## 4. Claudeの数値設計（今回確定した手法）

### 4-1. spawn_timeの逆算（向き合い型・待機3秒狙い）

```
護衛の交戦地到達時刻 = path index × 2.5s （護衛speed40, 1セル100px = 2.5s）
T_spawn = 護衛到達 - T_travel - 2(ghost) - 3(待機バッファ)
T_travel = spawn→leashTo直線距離(セル) × 100 ÷ 敵speed
```

例（交戦地1）: 護衛到達37.5s、spawn(9,13)→leashTo(9,7)=6セル、敵speed30
→ T_travel=20s、T_spawn=37.5-20-2-3=12.5s ✓

### 4-2. spawn位置はClaudeが自由に置く

小松さんの指定は「交戦地の位置」が絶対。**spawn位置は交戦地から逆算してClaudeが自由に決める**（後方に下げる等OK）。各spawnは交戦地から直線で届く位置に、6セル程度離して配置している。

### 4-3. タワーは「交戦地の真横」より「敵の直進経路に沿って」置く【重要知見】

交戦地4(7,15)で判明。敵はspawn(7,21)から(7,15)へ北上する。
- ✗ basic(8,14)＝交戦地の真横 → 敵の直進経路を射程に収める時間が短く撃ち負ける（1体着座・被弾）
- ○ basic(6,16)＝敵の北上経路沿い → 経路(7,18)(7,17)(7,16)を長く睨み、着座前に撃破（ノーダメ）

**タワー配置は交戦地座標だけ見るのではなく、敵がそこへ来る直進経路を睨む位置に置く。**

### 4-4. 確定パラメータ（このステージ）

- 護衛: speed 40 / hp 100
- 敵（向き合い型）: speed 30（護衛40より遅い）/ hp 150（salaryman baseの75をspawnEventで上書き）
- 待機バッファ: 3秒狙い（上限15秒フォールバックの範囲内）
- 敵種: 現状salarymanのみ（難易度上げる際にworker/police投入予定）

## 6. 隊列システム（CC実装完了）

### 6-1. 仕様（確定済み挙動）

spawnEventに `formation`（隊列体数）と `pitch`（追従間隔px）を追加：

```json
{ "time": 60.0, "spawn": "C", "enemy": "worker", "formation": 6, "pitch": 600, "speed": 30, "leashTo": {"col":18,"row":12} }
```

**挙動（実装済み）:**
1. **進軍**: 先頭はleashToへ直進。2体目以降は1つ前の個体をpitch距離で追従
2. **先頭撃破時の繰り上がり【核心】**: 先頭が倒れたら2体目が繰り上がってleashTo継承。「列を前から削る」パズル
3. **着座整列**: 先頭がleashTo到達で待機。後続は前の個体の後ろに縦列で詰めて整列待機
4. **順次解除**: 護衛が150px接近したら**先頭から順に**解除（一斉ではない）。先頭が護衛に取り付いてから2体目が動く

**実装ノート:**
- 後続移動は現在「直線追従」。`FormationGroup.USE_FLOW_FIELD = true` の1行で FlowField追従に切替可（本番マップ移行前に切替）
- 進軍中（着座前）は護衛と至近でも解除しない（交戦地交差はマップ設計側で回避）
- `formation`未指定のイベントは既存の`count+interval`動作を維持（後方互換）

### 6-2. 追従型JSON例（上辺・後方追走）

```json
{ "time": 5.0, "spawn": "REAR_START", "enemy": "salaryman",
  "formation": 3, "pitch": 300, "speed": 35, "hp": 100 }
```

※ leashTo なし = 最初からFlowFieldで護衛を追う（着座なし追従型）
