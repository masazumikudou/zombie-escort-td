# 仕様書04：特殊ゾンビ・敵ロードマップ（確定版 2026-06-12）

## 設計原則
1. **カウンター設計**：特殊ゾンビは「プレイヤーの特定戦術への回答」として作る。
   数字インフレ（ただ硬い・ただ速い、の量産）を避け、タワー・アイテムの
   使い分けを強制する存在にする
2. **生成コスト優先**：敵の見た目はGPT生成で量産するため、
   「乗ってる・浮いてる・転がってる」＝歩行サイクル不要のデザインを優先する。
   フレーム間の整合性問題を構造的に回避できる
3. **職業の表現はパーツエディタが本命**：ヘルメット＝工事、ネクタイ＝営業、
   エプロン＝店員等は既存のベース＋パーツ方式で量産。
   専用シルエットを起こすのは「ゲームの解き方が変わる敵」のみ
4. **耐性持ちは該当タワーの解放後にのみ投入**：プレイヤーが持っていない
   対策を咎めるのは理不尽。投入順はステージ進行と必ず同期させる

## 敵ロースター・実装ロードマップ

### 第1陣：ステータス差のみ（実装コスト極小）
| 敵 | 役割 | 仕様 | 生成メモ |
|---|---|---|---|
| ノーマルゾンビ群 | 基本 | 既存6種（色変え×素/ヘルメット/キャップ）。パーツ追加で随時増殖 | パーツエディタ運用 |
| おでぶゾンビ | タンク | 高HP・低速。ノックバック無効（ぐるぐるバット対策枠） | 歩行必要だがフレームブレが「揺れ」に見え破綻に強い |
| 電動キックボードゾンビ | 俊足 | 高速・低HP。単発系タワー（ゴムパチンコ）が間に合わない→連射の出番 | 乗り物系＝静止ポーズ1〜2枚、移動はコード |

### 第2陣：専用ロジック（中コスト）
| 敵 | 役割 | 仕様 |
|---|---|---|
| 鳥ゾンビ | 飛行・ルール破壊 | 遮蔽物無視で最短直進（A*不使用、直線移動）。地上トラップ（漫画）無効。近接（バット）無効。低HP・編隊スポーン（5羽並び等）。連射タワーの存在意義を作るウザ敵 |
| 解体業者ゾンビ | タワー齧り | タワーを最寄り目標としてA*（ゴール差し替えのみ）。低DPSでゆっくり齧る＝即時対応不要の「宿題」型。齧り中は無防備（他タワーの集中砲火で溶かす反撃の快感）。要：タワー耐久値の有効化、画面端のタワー被攻撃アラート＋タップでカメラジャンプ |
| 群衆ラッシュ | 物量演出 | 弱い通常種の大量湧き。寄り道ラッシュ生成器（仕様書01）を流用。独立モード（エンドレス等）への布石 |

### 第3陣：耐性持ち（該当タワー解放後のステージ限定）
| 敵 | 耐性 | 見た目の説得力 |
|---|---|---|
| レインコートゾンビ | スプリンクラー無効 | 雨具で水を弾く。一目で理解可能 |
| 活字嫌いゾンビ | 漫画ラック無効 | 漫画を一瞥して素通り |

### 第4陣：ボス級（章ボス・個別設計）
- 高耐久の名物ゾンビ。リレー間を持ち越して複数の護衛者にまたがって暴れる
- 1体ずつ専用に設計する贅沢品。ロードマップ上は最後
- 候補は将来の壁打ちで個別に詰める

### 候補プール（未確定・台帳）
- 買い物カートゾンビ：カート滑走、正面防御（カートが盾）。主婦の習性
- オフィスチェアゾンビ：キャスター椅子で滑走するサラリーマン
- ※どちらも乗り物系＝生成が楽。役割が第1〜2陣と被らないか精査してから採用

## カウンター対応表（設計の整合チェック用）
| プレイヤーの道具 | それを咎める敵 | その敵への解答 |
|---|---|---|
| ゴムパチンコ（単発重撃） | キックボード（速くて外す） | テニスマシン |
| テニスマシン（連射低威力） | おでぶ（豆鉄砲が通らない） | ゴムパチンコ／BBQ |
| ぐるぐるバット（近接） | 鳥（届かない）、おでぶ（ノックバック無効） | 遠距離タワー |
| スプリンクラー（スロー） | レインコート | 他のデバフ・火力 |
| 漫画ラック（停止） | 活字嫌い、鳥（踏まない） | オーラ型スロー・火力 |
| 置きっぱなし防衛網 | 解体業者（齧り） | アラート対応＋集中砲火 |

## stage JSONとの接続
- 敵はすべて type 文字列で指定（実装済みのtypeフィールドを使用）
- 種別ごとの基準値（HP/速度/ダメージ/報酬/特殊フラグ）はbalance.jsonに集約
- 特殊フラグ例：flying（遮蔽物無視）, knockbackImmune, waterImmune,
  mangaImmune, targetsTowers
