# 仕様書01：寄り道・誘導システム（確定版 2026-06-12）

## コンセプト
護衛者の導線が動的に変化する本作のコア機構。
- 強制寄り道：初見殺し＋学習体験（キングダムラッシュ式）
- 任意誘導：プレイヤーが看板で護衛を釣る。リスクリワードのやり込み軸
- 寄り道先での「滞在」＝固定地点の籠城戦。移動防衛と対になるプレイの切り替え
- 滞在中のゾンビラッシュ＝リスクの実体。報酬と対になる

## 世界観整合
護衛者はプレイヤーの存在を知らない（無自覚設定）。よってプレイヤーは
護衛者に直接指示できず、「物（看板等）を置いて釣る」間接誘導のみ可能。

## JSONスキーマ
```json
"detours": [{
  "id": "park_01",
  "type": "forced",            // forced（強制） | lureable（看板で誘発）
  "triggerCell": { "col": 8, "row": 7 },
  "site": { "col": 10, "row": 2 },
  "dwellTime": 12000,
  "reward": { "coins": 150, "gems": 0 },
  "rush": { "count": 15, "interval": 600, "spawns": [0, 2] }
}],
"lureSpots": [{ "col": 8, "row": 7 }]
```
- forced：護衛者がtriggerCellに接近すると必ず発動
- lureable：lureSpotsに看板が設置されている場合のみ、接近時に発動
- rushは省略可（ラッシュなしの安全な寄り道も作れる）
- rewardのgemsは恒久通貨。危険な寄り道に限定配置し周回動機にする

## 挙動フロー
1. 発動判定（forced：接近 / lureable：看板あり＋接近）
2. 予兆：護衛者の頭上に吹き出し（☕等）を3秒表示
3. ルート離脱、siteへA*で移動
4. 滞在（dwellTime）：護衛者は完全静止。座る等の演出
5. ラッシュ（定義時のみ）：滞在開始1〜2秒後に警告音＋対象スポーン口が
   赤点滅→ラッシュ開始
6. 滞在終了時に報酬獲得演出→元ルートへA*で復帰

## 誘導アイテム
- 初期実装は1種のみ：「タイムセール看板」
  - lureSpotsにのみ設置可能（タワー設置と同じポップアップ文法）
  - コスト制（コイン）。価格はバランス調整項目
  - 使い捨て（1回発動で消滅）
- 拡張候補（台帳・実装しない）：
  - コーヒースタンド：護衛者をその場でN秒停止（時間操作の戦術）
  - キャラ特性との連動（おばあちゃんは看板無視、など→仕様書02参照）

## 設計原則
- 寄り道は「読めないが、直前に読める」：予兆3秒で緊急対応の面白さに変換
- 任意寄り道の報酬は実力で取りに行くボーナス：
  報酬（コイン/ジェム）⇔ リスク（滞在ラッシュ＋カバー薄い地形）
- lureSpotsの配置はマップデザイン：本ルート近くは安全で報酬小、
  遠くは危険で報酬大、を地形で表現する

## エディタ要件
- lureSpotsモード（buildSpotsと同じ操作感）
- detour定義UI：trigger/site/dwell/reward/rushをフォーム入力
- ※detour定義UIが重い場合、初期はJSON手書きでも可（項目が少ないため）

## 拡張の置き場（実装対象外）
- ゾンビラッシュ生成器は独立モード（エンドレス/パニックモード）に流用可能
